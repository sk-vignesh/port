# Plan 01-01: Apply Migration 009 & Smoke-Test Edge Functions

## Objective
Apply migration 009 (asset_class on portfolios) to the production Supabase instance, and smoke-test all 4 Edge Functions to confirm they are deployed and returning valid responses.

## Context
Migration 009 is the most critical unblocked fix — it enables asset class grouping across Holdings, Dashboard, and Portfolio pages. It is safe to apply (NOT NULL with DEFAULT 'EQUITY' — no data loss). All 4 Edge Functions (gains, personal-index, import-trades, price-search) exist but have never been verified against production.

### Task 2: Apply Migration 009 in Supabase SQL Editor

<read_first>
- `supabase/migrations/009_asset_classes.sql`
</read_first>

<action>
Open Supabase SQL Editor and run the full contents of `supabase/migrations/009_asset_classes.sql`:

```sql
-- 1. Add asset_class to portfolios
ALTER TABLE portfolios
  ADD COLUMN IF NOT EXISTS asset_class TEXT NOT NULL DEFAULT 'EQUITY';

-- 2. Tag the 4 default portfolios by name
UPDATE portfolios SET asset_class = 'EQUITY'       WHERE name = 'Stocks';
UPDATE portfolios SET asset_class = 'COMMODITY'    WHERE name = 'Commodities';
UPDATE portfolios SET asset_class = 'FIXED_INCOME' WHERE name = 'Fixed Income';
UPDATE portfolios SET asset_class = 'REAL_ESTATE'  WHERE name = 'Real Estate';

-- 3. Add optional asset-class metadata columns to portfolio_transactions
ALTER TABLE portfolio_transactions ADD COLUMN IF NOT EXISTS unit_type TEXT DEFAULT 'SHARES';
ALTER TABLE portfolio_transactions ADD COLUMN IF NOT EXISTS face_value BIGINT;
ALTER TABLE portfolio_transactions ADD COLUMN IF NOT EXISTS coupon_rate NUMERIC(8, 4);
ALTER TABLE portfolio_transactions ADD COLUMN IF NOT EXISTS maturity_date DATE;
ALTER TABLE portfolio_transactions ADD COLUMN IF NOT EXISTS interest_frequency TEXT;
ALTER TABLE portfolio_transactions ADD COLUMN IF NOT EXISTS area INTEGER;

-- 4. Backfill unit_type
UPDATE portfolio_transactions SET unit_type = 'SHARES' WHERE unit_type IS NULL;
```

After running, verify with:
```sql
SELECT id, name, asset_class FROM portfolios LIMIT 10;
```
</action>

<acceptance_criteria>
- No SQL errors when running migration 009
- `SELECT column_name FROM information_schema.columns WHERE table_name = 'portfolios' AND column_name = 'asset_class'` returns 1 row
- `SELECT DISTINCT asset_class FROM portfolios` returns values including 'EQUITY'
- `SELECT column_name FROM information_schema.columns WHERE table_name = 'portfolio_transactions' AND column_name = 'unit_type'` returns 1 row
</acceptance_criteria>

### Task 3: Smoke-Test All 4 Edge Functions

<read_first>
- `supabase/functions/gains/index.ts`
- `.env.local` (NEXT_PUBLIC_SUPABASE_URL and NEXT_PUBLIC_SUPABASE_ANON_KEY)
</read_first>

<action>
For each Edge Function, make a test HTTP request using curl or the browser. The Supabase project URL is: `https://gspnjzckdlkhidlivrzk.supabase.co`

The anon key is `NEXT_PUBLIC_SUPABASE_ANON_KEY` from `.env.local`.

Run in PowerShell:
```powershell
$SUPABASE_URL = "https://gspnjzckdlkhidlivrzk.supabase.co"
$ANON_KEY = (Get-Content .env.local | Where-Object { $_ -match "NEXT_PUBLIC_SUPABASE_ANON_KEY" }) -replace ".*=", ""

# Test gains function (requires auth — will return 401 if deployed, error if not deployed)
Invoke-WebRequest -Uri "$SUPABASE_URL/functions/v1/gains" -Headers @{ "Authorization" = "Bearer $ANON_KEY" } | Select-Object StatusCode

# Test price-search function (no auth required)
Invoke-WebRequest -Uri "$SUPABASE_URL/functions/v1/price-search?q=reliance" -Headers @{ "apikey" = "$ANON_KEY" } | Select-Object StatusCode, Content
```

For each function, record:
- HTTP 200 or 401 = deployed ✓
- HTTP 404 or connection error = NOT deployed ✗

If any function is NOT deployed, deploy it:
```powershell
npx supabase functions deploy gains --project-ref gspnjzckdlkhidlivrzk
npx supabase functions deploy personal-index --project-ref gspnjzckdlkhidlivrzk
npx supabase functions deploy import-trades --project-ref gspnjzckdlkhidlivrzk
npx supabase functions deploy price-search --project-ref gspnjzckdlkhidlivrzk
```
</action>

<acceptance_criteria>
- All 4 Edge Function URLs return HTTP 200 or 401 (not 404 or network error)
- `price-search?q=reliance` returns a JSON array (may be empty)
- If deployment was needed, `supabase functions deploy` exits with code 0
</acceptance_criteria>
