# Plan 01-02: Apply Migration 010 & Remove TypeScript `unknown` Casts

## Objective
Apply migration 010 (classification_id FK on portfolios) after verifying its prerequisite table exists. Then clean up all TypeScript `unknown` casts that were needed as workarounds before migration 009 was applied.

## Context
Migration 010 references a `classifications` table via a FK. This table may or may not exist depending on prior manual migrations. We must check before applying. After migration 009 is applied (Plan 01-01), the `asset_class` column exists in the database and the TypeScript `unknown` casts throughout the app can be removed and replaced with proper typed interfaces.

---

## Tasks

### Task 1: Verify `classifications` Table Exists Before Applying Migration 010

<read_first>
- `supabase/migrations/010_portfolio_segments.sql`
</read_first>

<action>
In Supabase SQL Editor, run:

```sql
SELECT table_name FROM information_schema.tables 
WHERE table_schema = 'public' AND table_name = 'classifications';
```

**If result has 1 row** → classifications table exists → proceed to Task 2.

**If result is empty** → classifications table does NOT exist → migration 010 will fail with FK error. 

In that case, create the table first:
```sql
CREATE TABLE IF NOT EXISTS classifications (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id     UUID NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE,
  name        TEXT NOT NULL,
  color       TEXT,
  created_at  TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
ALTER TABLE classifications ENABLE ROW LEVEL SECURITY;
CREATE POLICY "Users can CRUD own classifications" ON classifications
  USING (auth.uid() = user_id) WITH CHECK (auth.uid() = user_id);
```
Then proceed to Task 2.
</action>

<acceptance_criteria>
- `SELECT table_name FROM information_schema.tables WHERE table_schema = 'public' AND table_name = 'classifications'` returns 1 row (either pre-existing or just created)
</acceptance_criteria>

### Task 2: Apply Migration 010

<read_first>
- `supabase/migrations/010_portfolio_segments.sql`
</read_first>

<action>

```sql
ALTER TABLE portfolios
```

</action>

### Task 3: Remove `unknown` Casts in `app/(app)/portfolios/page.tsx`

<read_first>
- `app/(app)/portfolios/page.tsx`
- `lib/assetClasses.ts`
</read_first>

<action>
The file currently uses patterns like:
```ts
const portfoliosRaw = data ?? []
const portfolios = (portfoliosRaw as unknown as Portfolio[]) ?? []
```

Add a proper local interface at the top of the file (after imports):
```ts
interface Portfolio {
  id: string
  name: string
  asset_class: string
  is_retired: boolean
  created_at: string
}
```

Then replace all `as unknown as Portfolio[]` casts with direct typing:
```ts
const portfolios: Portfolio[] = (data ?? []) as Portfolio[]
```

Do the same for any `PortfolioWithStats` or similar derived interfaces used in that file.
</action>

<acceptance_criteria>
- `app/(app)/portfolios/page.tsx` contains `interface Portfolio {` with `asset_class: string`
- `app/(app)/portfolios/page.tsx` does NOT contain the string `as unknown as`
- `npx tsc --noEmit` exits with code 0
</acceptance_criteria>
