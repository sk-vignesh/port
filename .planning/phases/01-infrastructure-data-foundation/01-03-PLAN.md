# Plan 01-03: Loading Skeletons & Live Security Search

## Objective
Add `loading.tsx` skeleton screens to all major server-rendered pages to eliminate blank-page flashes. Update `SecuritySearchInput` to query the `price-search` Edge Function instead of the hardcoded 14 KB static NSE list.

## Context
Both tasks are independent frontend improvements (wave 3 — no dependency on migrations). The skeleton screens use Next.js App Router's `loading.tsx` convention. The SecuritySearchInput currently imports a 14 KB hardcoded list from `lib/indian-stocks.ts` — replacing it with a live query reduces bundle size and keeps results current.

### Task 6: Update SecuritySearchInput to Use Live price-search Edge Function

<read_first>
- `components/SecuritySearchInput.tsx` (full file — understand current structure)
- `lib/indian-stocks.ts` (first 20 lines — understand the IndianStock interface)
- `.env.local` (NEXT_PUBLIC_SUPABASE_URL)
</read_first>

<action>
Rewrite `SecuritySearchInput.tsx` to query the `price-search` Edge Function instead of the static list. Keep the same UI/UX (dropdown, keyboard nav, sector colors).

Key changes:
1. Remove `import NSE_STOCKS, { type IndianStock } from '@/lib/indian-stocks'`
2. Add local `SearchResult` type:
   ```ts
   interface SearchResult {
     symbol: string
     name: string
     sector?: string
   }
   ```
3. Replace the `useMemo` results with `useState<SearchResult[]>([])` and a `useEffect` that debounces (300ms) and calls the Edge Function:
   ```ts
   const SUPABASE_URL = process.env.NEXT_PUBLIC_SUPABASE_URL
   const ANON_KEY = process.env.NEXT_PUBLIC_SUPABASE_ANON_KEY

   useEffect(() => {
     if (query.trim().length < 2) { setResults([]); return }
     const timer = setTimeout(async () => {
       setLoading(true)
       try {
         const res = await fetch(
           `${SUPABASE_URL}/functions/v1/price-search?q=${encodeURIComponent(query.trim())}`,
           { headers: { 'apikey': ANON_KEY ?? '' } }
         )
         if (res.ok) {
           const data = await res.json()
           setResults(Array.isArray(data) ? data.slice(0, 9) : [])
         }
       } catch { setResults([]) }
       finally { setLoading(false) }
     }, 300)
     return () => clearTimeout(timer)
   }, [query])
   ```
4. Add `const [loading, setLoading] = useState(false)` state
5. Show a loading indicator in the dropdown while fetching (spinner or "Searching…" text)
6. Keep the same `onSelect` prop interface but use `SearchResult` type
7. Update `Props.onSelect` type to `(result: SearchResult) => void`
8. Update all callers of `SecuritySearchInput` that used `IndianStock` type to use `SearchResult`

**Fallback:** If Edge Function returns non-200, show "Search unavailable" message in dropdown.
</action>

<acceptance_criteria>
