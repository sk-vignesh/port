# Plan 02-01: Extend SearchResult + UnifiedTradeSearch Modal

## Objective
Extend the `SearchResult` type with `currency` and `exchange` fields (already returned by `price-search` but not passed through). Build a new `UnifiedTradeSearch` modal component that implements the 2-step flow: (1) search for a security, (2) pick which portfolio/asset class to trade in — then navigate to the pre-populated trade form.

## Context
The `price-search` Edge Function already returns `{ symbol, name, exchange, currency, type }` from Yahoo Finance. The rewritten `SecuritySearchInput` (Phase 1, Wave 3) uses this function but discards `currency` and `exchange`. The trade form URL supports `?security_id=` pre-population via `EquityTradeForm`'s `urlSecurityId` param. Missing piece: a modal that glues search → portfolio selection → trade form navigation.

const { data: existing } = await supabase
  .from('securities')
  .select('id')
  .eq('ticker_symbol', result.symbol)
  .maybeSingle()

if (existing) {
  router.push(`/portfolios/${portfolio.id}/transactions/new?security_id=${existing.id}`)
} else {
  // Create security then navigate
  const { data: { user } } = await supabase.auth.getUser()

}
