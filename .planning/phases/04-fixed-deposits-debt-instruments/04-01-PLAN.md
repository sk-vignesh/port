# Plan 04-01: DB Migration + FD Accrued Value Math Library

## Objective
Create migration 012 that adds the 3 columns the existing `FixedIncomeTradeForm` already writes to (`face_value`, `coupon_rate`, `interest_frequency`). Build a pure-function `lib/fd.ts` library that computes FD accrued value, projected maturity value, and days-to-maturity given a transaction row — used on the portfolio detail and Holdings pages.

## Context
`FixedIncomeTradeForm` already inserts `face_value`, `coupon_rate`, `maturity_date`, `interest_frequency` into `portfolio_transactions`. These columns don't exist yet in any migration. This is why every Fixed Income purchase currently fails with a Postgres error. The form is complete; only the DB schema is missing.

### AMFI-style compounding formulas for FDs

```
Simple interest (RD, NSC, some FDs):
  current = principal × (1 + rate/100 × days/365)

Compound interest (standard bank FD):
  quarterly: current = principal × (1 + rate/400)^(4 × years)
  annual:    current = principal × (1 + rate/100)^years

Maturity value:
  projected = principal × (1 + rate/frequency_divisor)^(frequency_divisor × years)
  where frequency_divisor: MONTHLY=12, QUARTERLY=4, SEMI_ANNUAL=2, ANNUAL=1
```

### Task 2: Create lib/fd.ts — FD Math Library

<action>
Create `lib/fd.ts`:

```ts
/**
 * Fixed Income / FD math utilities
 *
 * All monetary inputs/outputs use BIGINT × 100 encoding (same as the rest of the codebase).
 * Rate inputs are plain percentages (e.g. 7.5 = 7.5% p.a.).
 */

export interface FDTransaction {
  /** Purchase date (ISO string) */
  date: string
  /** Maturity date (ISO string), null for perpetual instruments */
  maturity_date: string | null
  /** Principal in raw BIGINT×100 units */
  face_value: number | null
  /** Annual rate in percent, e.g. 7.5 */
  coupon_rate: number | null
  /** Payout frequency */
  interest_frequency: 'MONTHLY' | 'QUARTERLY' | 'SEMI_ANNUAL' | 'ANNUAL' | 'AT_MATURITY' | null
  /** Transaction type — only BUY rows are used for FD math */
  type: string
}

const FREQ_N: Record<string, number> = {
  MONTHLY: 12, QUARTERLY: 4, SEMI_ANNUAL: 2, ANNUAL: 1, AT_MATURITY: 1,
}

/** Days between two ISO date strings */
export function daysBetween(from: string, to: string): number {
  return Math.floor((new Date(to).getTime() - new Date(from).getTime()) / 86_400_000)
}

/** Days until maturity from today (negative = already matured) */
export function daysToMaturity(maturityDate: string): number {
  return daysBetween(new Date().toISOString().slice(0, 10), maturityDate)
}

/**
 * Current accrued value of an FD (compound interest model).
 * Returns value in BIGINT×100 units.
 * Returns face_value if rate / maturity are missing.
 */
export function fdAccruedValue(tx: FDTransaction, asOf = new Date().toISOString().slice(0, 10)): number {
  const principal = tx.face_value ?? 0
  if (!principal || !tx.coupon_rate || tx.coupon_rate <= 0) return principal

  const n    = FREQ_N[tx.interest_frequency ?? 'AT_MATURITY'] ?? 1
  const rate = tx.coupon_rate / 100
  const days = daysBetween(tx.date, asOf)
  if (days <= 0) return principal

  const years  = days / 365
  const accrued = principal * Math.pow(1 + rate / n, n * years)
  return Math.round(accrued)
}

/**
 * Projected maturity value of an FD.
 * Returns value in BIGINT×100 units.
 */
export function fdMaturityValue(tx: FDTransaction): number {
  const principal = tx.face_value ?? 0
  if (!principal || !tx.coupon_rate || !tx.maturity_date) return principal

  const n     = FREQ_N[tx.interest_frequency ?? 'AT_MATURITY'] ?? 1
  const rate  = tx.coupon_rate / 100
  const years = daysBetween(tx.date, tx.maturity_date) / 365
  if (years <= 0) return principal

  return Math.round(principal * Math.pow(1 + rate / n, n * years))
}

/**
 * Annual interest payout amount for a given FD (face_value × rate / freq).
 * Returns value in BIGINT×100 units.
 */
export function fdPeriodicInterest(tx: FDTransaction): number {
  const principal = tx.face_value ?? 0
  if (!principal || !tx.coupon_rate) return 0
  const n = FREQ_N[tx.interest_frequency ?? 'ANNUAL'] ?? 1
  return Math.round(principal * (tx.coupon_rate / 100) / n)
}
```
