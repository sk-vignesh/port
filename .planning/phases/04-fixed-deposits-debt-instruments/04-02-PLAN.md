# Plan 04-02: FD Card View + Fixed Income Holdings Display

## Objective
Build a `FDCard` component that shows an FD's current accrued value, projected maturity value, days to maturity, and coupon rate — using the `lib/fd.ts` math library. Wire it into the portfolio detail page so FIXED_INCOME portfolios display FD cards instead of the standard AG Grid holdings table.

## Context
The portfolio detail page (`portfolios/[id]/page.tsx`) currently shows all transactions via `PortfolioDetailClient` / `PortfolioTransactionsGrid`. For FIXED_INCOME portfolios, we want to also show a "FD Summary" section at the top — one card per active FD instrument showing its live accrued value and maturity status.

The `lib/fd.ts` library (built in 04-01) provides the compound interest math. The portfolio page already fetches all transactions and the portfolio's `asset_class`.

<read_first>
- `lib/fd.ts` (just built in 04-01 — all math functions available)
- `lib/format.ts` (for formatAmount)
</read_first>

<action>
Create `components/FDCard.tsx` — a server-importable component that computes and renders one FD's live status.

```tsx
// Server component — no 'use client' needed
import { fdAccruedValue, fdMaturityValue, daysToMaturity, fdPeriodicInterest } from '@/lib/fd'
import { formatAmount } from '@/lib/format'
import type { FDTransaction } from '@/lib/fd'

interface Props {
  tx: FDTransaction
  currency: string
}

export default function FDCard({ tx, currency }: Props) {
  const accrued    = fdAccruedValue(tx)
  const maturity   = fdMaturityValue(tx)
  const principal  = tx.face_value ?? 0
  const gain       = accrued - principal
  const gainPct    = principal > 0 ? (gain / principal) * 100 : 0
  const dtm        = tx.maturity_date ? daysToMaturity(tx.maturity_date) : null
  const periodic   = fdPeriodicInterest(tx)

  const isMatured  = dtm !== null && dtm <= 0
  const isSoon     = dtm !== null && dtm > 0 && dtm <= 30

  return (
    <div style={{
      padding: 20, borderRadius: 12,
      border: `1px solid ${isMatured ? 'rgba(245,158,11,0.4)' : isSoon ? 'rgba(239,68,68,0.3)' : 'var(--color-border)'}`,
      background: isMatured ? 'rgba(245,158,11,0.06)' : 'var(--color-bg-elevated)',
    }}>
      {/* Header */}
      <div style={{ display: 'flex', justifyContent: 'space-between', alignItems: 'flex-start', marginBottom: 16 }}>
        <div>
          ... (instrument name, bank, rate badge)
        </div>
        <div ... >{isMatured ? '⚠ Matured' : isSoon ? `⚡ ${dtm}d left` : dtm ? `${dtm}d to maturity` : null}</div>
      </div>

      {/* Value grid */}
      <div style={{ display: 'grid', gridTemplateColumns: 'repeat(3,1fr)', gap: 16 }}>
        Principal, Current Value, Maturity Value
      </div>

      {/* Progress bar: elapsed / total term */}
    </div>
  )
}
```

**Implementation notes:**
- Show a progress bar using CSS: `elapsed / total term × 100%` using `daysBetween(tx.date, today) / daysBetween(tx.date, tx.maturity_date) * 100`
- Label rows: Principal, Current Accrued Value (green), Projected Maturity Value
- Show coupon rate badge: `7.50% p.a. · Quarterly`
- Color code status: amber border if matured, red-tinted if ≤30 days, normal otherwise
- Keep it as a **server component** (no `'use client'`) since it uses the math library with no interactivity needed
</action>
