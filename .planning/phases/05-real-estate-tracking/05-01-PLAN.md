# Plan 05-01: DB Migration + Property Valuation Card

## Objective
Create migration 013 that adds the `area` column and `portfolio_transaction_units` table that `RealEstateTradeForm` already writes to (currently failing silently — same pattern as Phase 4). Build a `PropertyCard` server component that displays a real estate holding's purchase cost, current value (manual override), rental yield, and price-per-sqft — rendered on the REAL_ESTATE portfolio detail page.

## Context
`RealEstateTradeForm` is already fully built (206 lines). It writes to:
1. `portfolio_transactions.area` — built-up area in sq ft
2. `portfolio_transaction_units` table — for stamp duty / registration fees

Neither exists in the database yet. This is the critical fix. After migration 013 is applied:
- All Real Estate purchase entries work end-to-end
- Stamp duty is correctly recorded as a fee unit

### Property Valuation Logic
Unlike FDs (which we compute via compound interest), real estate values are **user-provided estimates**. The display shows:
- **Purchase cost** — from the original BUY transaction amount + stamp duty from transaction units
- **Current value** — from the latest `security_prices` entry (manual update via the securities page) OR fallback to purchase cost
- **Price per sq ft** — `current_value / area`
- **Rental yield** — `(annual_rental_income / purchase_cost) × 100`
- **Capital gain** — `current_value - total_cost`

### Task 1: Database Migration 013

<action>
Create `supabase/migrations/013_real_estate.sql`:

```sql
-- Migration 013: Real Estate columns and fee tracking table

-- Add area column to portfolio_transactions (for SQFT tracking)
ALTER TABLE portfolio_transactions
  ADD COLUMN IF NOT EXISTS area INTEGER;  -- built-up area in square feet

COMMENT ON COLUMN portfolio_transactions.area IS 'Built-up/carpet area in square feet (used for real estate transactions)';

-- Create portfolio_transaction_units table for fees, taxes, and sub-components
CREATE TABLE IF NOT EXISTS portfolio_transaction_units (
  id             UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  transaction_id UUID NOT NULL REFERENCES portfolio_transactions(id) ON DELETE CASCADE,
  type           TEXT NOT NULL,          -- FEE, TAX, BROKERAGE, INTEREST, DIVIDEND
  amount         BIGINT NOT NULL,        -- × 100 (same encoding as amount)
  currency_code  TEXT NOT NULL DEFAULT 'INR',
  note           TEXT,
  created_at     TIMESTAMPTZ DEFAULT NOW()
);

CREATE INDEX IF NOT EXISTS idx_ptu_transaction_id
  ON portfolio_transaction_units (transaction_id);

COMMENT ON TABLE  portfolio_transaction_units IS 'Fee, tax, and sub-component breakdown for a portfolio transaction (e.g. stamp duty, brokerage)';
COMMENT ON COLUMN portfolio_transaction_units.type IS 'Unit type: FEE, TAX, BROKERAGE, INTEREST, DIVIDEND';
```
</action>

### Task 2: Build PropertyCard Component

<action>
Create `components/PropertyCard.tsx` — a server-compatible component (no `'use client'`) that renders one property's investment summary.

**Props:**
```ts
interface Props {
  name:          string         // property security name
  securityId:    string         // for the "Update Valuation" link
  purchaseCost:  number         // BIGINT×100 — sum of BUY transactions
  currentValue:  number | null  // BIGINT×100 — from security_prices, or null
  stampDuty:     number         // BIGINT×100 — from transaction_units FEE rows
  area:          number | null  // sq ft
  annualRental:  number         // BIGINT×100 — sum of INTEREST transactions for current year
  currency?:     string
}
```

**Display elements:**
1. **Property name + type badge** (residential / commercial / land etc, extracted from security name)
2. **Value summary strip**: Purchase Cost · Current Value (or "Market price not set") · Capital Gain/Loss
3. **Metrics grid**:
   - Price / sq ft: `current_value / area` (if both available)
   - Rental yield: `(annualRental / purchaseCost) × 100`%
   - Total cost: `purchaseCost + stampDuty`
4. **CTA**: "Update Valuation" link → `/securities/{id}` (for manual current price entry)
5. **Color coding**: Green gain, red loss; amber for "Market price not set"
</action>
