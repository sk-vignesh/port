# Plan 07-01: Watch Alerts API + In-App Alert Badge

## Objective
1. Verify and harden the `/api/watch-alerts` REST endpoints (POST + PATCH + DELETE)
2. Add an in-app alert badge to the sidebar navigation that lights up when any triggered watch alert exists — checked on page load from the `watch_alerts` table

## Context

### WatchlistCards (already complete — DO NOT rebuild it)
`WatchlistCards.tsx` is a 196-line fully-implemented client component:
- `AlertBadge`: renders triggered/active state with icons
- `AlertForm`: inline collapsible form — PRICE_ABOVE/BELOW/CHANGE_PCT_UP/DOWN types, threshold, note
- Calls `POST /api/watch-alerts` (new alert) and `PATCH /api/watch-alerts/{id}` (update)
- **Already wired into the watchlist page** — nothing to change in WatchlistCards or watchlists/page.tsx

### Alert checking logic
The `watch_alerts` table has `triggered_at TIMESTAMPTZ` — set when the condition is last met. For Phase 7, "trigger checking" is simple: compare the alert threshold against the latest price in `security_prices` (or `price_history`) at page load. If threshold met and `triggered_at IS NULL`, update the record server-side.

For now, the alert badge will use a simple approach: the sidebar server component queries `watch_alerts` where `triggered_at IS NOT NULL AND is_active = true` — these are already-triggered alerts the user hasn't acknowledged. This is fast, zero-latency, and doesn't require any edge function.

<action>
Add a red notification dot/badge to the "Watchlists" nav item in the Sidebar when there are triggered alerts.

**Implementation approach** (server component, zero client JS):

In `app/(app)/layout.tsx` (or wherever the `Sidebar` is rendered from the server):
1. Fetch the triggered alert count:
```ts
const supabase = await createClient()
const { data: { user } } = await supabase.auth.getUser()
let triggeredAlertCount = 0
if (user) {
  const { count } = await supabase
    .from('watch_alerts')
    .select('*', { count: 'exact', head: true })
    .eq('user_id', user.id)
    .eq('is_active', true)
    .not('triggered_at', 'is', null)
  triggeredAlertCount = count ?? 0
}
```

2. Pass `triggeredAlertCount` as a prop to Sidebar:
```tsx
<Sidebar triggeredAlertCount={triggeredAlertCount} />
```

3. In `Sidebar.tsx`, accept the prop and render a small red dot on the Watchlists nav item:
```tsx
{triggeredAlertCount > 0 && (
  <span style={{
    display: 'inline-flex', alignItems: 'center', justifyContent: 'center',
    width: 18, height: 18, borderRadius: 999,
    background: '#ef4444', color: '#fff',
    fontSize: '0.6rem', fontWeight: 800,
    marginLeft: 6,
  }}>
    {triggeredAlertCount > 9 ? '9+' : triggeredAlertCount}
  </span>
)}
```
</action>
