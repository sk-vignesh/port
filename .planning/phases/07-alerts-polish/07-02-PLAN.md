# Plan 07-02: Broker Tagging + Debug File Cleanup

## Objective
1. Add `broker_name TEXT` column to `portfolios` and `accounts` tables (display-only label, e.g. "Zerodha", "Groww", "HDFC Securities")
2. Surface the broker name on portfolio and account edit forms (simple text input, no search UX needed)
3. Display broker name as a badge in the portfolio detail page header and the accounts list
4. Clean up root-level debug artifacts: `b.txt`, any `build_err*.txt` files

---

## Tasks

### Task 1: Database Migration 014 — broker_name column

<action>
Create `supabase/migrations/014_broker_tag.sql`:

```sql
-- Migration 014: Broker tagging (display-only, no FK enforcement)
ALTER TABLE portfolios ADD COLUMN IF NOT EXISTS broker_name TEXT;
ALTER TABLE accounts   ADD COLUMN IF NOT EXISTS broker_name TEXT;

COMMENT ON COLUMN portfolios.broker_name IS 'Display label for the brokerage (e.g. Zerodha, Groww, HDFC Securities)';
COMMENT ON COLUMN accounts.broker_name   IS 'Display label for the brokerage or bank account provider';
```
</action>

<acceptance_criteria>
- `supabase/migrations/014_broker_tag.sql` exists and is idempotent
</acceptance_criteria>
