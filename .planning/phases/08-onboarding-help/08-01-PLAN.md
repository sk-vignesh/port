# Plan 08-01: Revamped Onboarding Wizard

## Objective
Replace the existing 3-step portfolio-creation wizard with a full 5-slide onboarding experience that:
- **Blocks entry** — shown as a fullscreen modal on first login (before the regular app layout loads), cannot be skipped
- **Educates** — 2 educational slides showing value proposition and feature highlights
- **Guides** — 3 import-path slides (CAS PDF / CSV / Manual), each self-contained
- **Uses defaults** — manual path auto-uses the user's default Stocks portfolio + Cash INR account; user just picks a stock and quantity

## Context

### Trigger Logic
The onboarding should show when `user_settings` has no `onboarding_completed` flag. After completing any path, set `onboarding_completed = true` in `user_settings`.

Add column: `ALTER TABLE user_settings ADD COLUMN IF NOT EXISTS onboarding_completed BOOLEAN NOT NULL DEFAULT false;`

This lives in migration **016_onboarding_flag.sql**.

In `app/(app)/layout.tsx`, fetch this flag server-side. If `false` → render `<OnboardingModal />` overlaid on top. User cannot dismiss until they complete any path.

## Task Breakdown

### Task 1: Migration 016 — onboarding_completed flag
```sql
ALTER TABLE user_settings ADD COLUMN IF NOT EXISTS onboarding_completed BOOLEAN NOT NULL DEFAULT false;
```

### Task 2: Build `OnboardingModal` component structure

Create `components/onboarding/OnboardingModal.tsx` — a fullscreen overlay (z-index: 9999) with:
- Slide state machine (0–4)
- Progress dots at top
- Animated slide transitions (opacity + translateY)
- `onComplete()` callback that patches `user_settings.onboarding_completed = true` then unmounts

### Task 3: Build each slide component

`components/onboarding/slides/`:
- `Slide1Welcome.tsx` — Logo + tagline + animated asset icons
- `Slide2Features.tsx` — 4 feature cards
- `Slide3ChoosePath.tsx` — 3 path choice cards
- `Slide4Manual.tsx` — stock search + qty + date + price, uses SmartSecuritySearch
- `Slide5Done.tsx` — success summary

### Task 4: Wire into layout

In `app/(app)/layout.tsx`:
1. Fetch `onboarding_completed` from `user_settings`
2. Pass `showOnboarding={!completed}` to a `ClientOnboardingGate` client component
3. `ClientOnboardingGate` conditionally renders `<OnboardingModal />` as a portal

### Task 5: Mark complete via API

`PATCH /api/onboarding-complete` — sets `user_settings.onboarding_completed = true`
Called automatically when user reaches Slide 5 or completes any import path.
